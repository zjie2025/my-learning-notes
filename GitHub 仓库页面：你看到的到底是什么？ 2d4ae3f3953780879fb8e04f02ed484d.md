# GitHub 仓库页面：你看到的到底是什么？

当你打开一个仓库（比如 `learning-notes` 或 `liubc-app`），顶部通常有这些按钮：

```
Code |Issues |Pullrequests |Actions |Projects |Wiki |Security |Insights

```

👉 **你现在真正需要理解的只有 3 个：**

- ✅ Code
- ✅ Issues
- ✅ History（它在 Code 里面）

Pull requests 我也会讲，但先当“了解即可”。

---

# 一、Code（最重要）

## 🧠 Code = “文件 + 当前状态”

你现在 **99% 的时间都在用这个**。

### Code 里你能做什么？

- 看所有文件（index.html / pdf / js / css）
- 上传文件
- 修改文件
- 删除文件
- 看最新版本

👉 **一句话：Code 就是这个项目现在长什么样**

---

### Code 页面里的几个常见按钮

### 1️⃣ Add file

- Upload files（上传文件）
- Create new file（新建文件）

你之前用它来：

- 上传 PDF
- 新建 `pdf/2025/` 文件夹
- 新建 `pdf-year.html`

✅ 非常重要，新手最常用。

---

### 2️⃣ Code（绿色按钮）

- 下载整个项目（ZIP）
- 给别人分享代码
- 以后学到“命令行”才会用到

👉 现在你可以 **忽略它**，知道存在就行。

---

### 3️⃣ 文件名 / 文件夹

- 点文件 → 看内容
- 点 ✏️ → 编辑
- 改路径 → 移动文件（你已经用过）

---

# 二、History（历史记录）——其实你已经用过

History 不在顶栏，它在 **每个文件里**。

### 怎么找到？

1. 点开一个文件（比如 `index.html`）
2. 右上角有一个 **History**

---

### History 是干嘛的？

👉 **它是这个文件的“时间机器”**

你可以看到：

- 什么时候改的
- 改了什么
- 改之前长什么样

📌 如果哪天你把网站“改坏了”：

- 不慌
- 打开 History
- 回到昨天的版本

这是 GitHub **最值钱的功能之一**。

---

# 三、Issues（问题 / 备忘 / 讨论）

## 🧠 Issues = “事情记录本”

不是“报错”那么狭隘。

### 对你来说，Issues 可以用来：

- 记学习问题
- 记以后要改的事
- 教会同工留言

### 示例（你可以这么用）：

- ❓「在线圣经按钮手机打不开」
- 📝「下周要上传 2025-12-29 周报」
- 💡「以后想加奉献功能」

---

### Issues 怎么用？

1. 点 **Issues**
2. 点 **New issue**
3. 写标题 + 内容
4. Submit

👉 就像一个**长期不会丢的记事本**

---

### 好处

- 不用写在纸上
- 不用怕忘
- 有时间顺序
- 可以标记已完成（Close）

---

# 四、Pull Requests（了解即可）

## 🧠 Pull Request = “请求合并改动”

这是**多人合作**才常用。

对你现在来说：

- ❌ 不必须
- ❌ 不常用
- ✅ 知道名字就够

### 简单理解：

> 「我在别的地方改了点东西，
> 
> 
> 现在想合并回主版本」
> 

你一个人维护网站时：

- 基本用不到
- 以后有同工一起写代码才会用

---

# 五、Actions / Projects / Wiki（暂时不用）

你现在可以直接这样记：

- Actions：自动化（先不碰）
- Projects：任务看板（以后再说）
- Wiki：另一种写文档方式（你现在用 README + Notion 更合适）

---

# 六、给你一个“记忆口诀”

你只记住这三句就够：

> Code：现在的样子
> 
> 
> **History**：过去的样子
> 
> **Issues**：要做和想做的事
> 

Pull Requests = 多人合作时再说。

---

# 七、你可以马上做的一个小练习（5 分钟）

👉 在 `learning-notes` 仓库里：

1. 打开 **Issues**
2. 新建一个 Issue，标题写：
    
    ```
    我的第一个 GitHub Issue
    
    ```
    
3. 内容写：
    
    ```
    今天学会了 GitHub 的Code / Issues / History
    
    ```
    
- `index.md` = **第一级（首页）**
- `notes.md`、`bible.md`、`church.md`… = **第二级（首页下面的栏目页）**
- 每篇具体笔记（比如 `2025-01-15-hebrews-ch1.md`）= **更下一级**（由栏目页去链接）

你要做的就是：**把 `bible.md` 创建出来，然后在 `index.md` 里加一条链接指向它**。

---

## 1）创建 `bible.md`（第二级页面）

在仓库首页：

1. **Add file → Create new file**
2. 文件名：`bible.md`
3. 内容先用这个模板（可直接复制）：

```markdown
# 📖 Bible 学习

## 目录
- [希伯来书](./hebrews.md)
- [解经与应用](./exegesis.md)

```

1. **Commit changes**

> 你也可以先只写一行“以后再补”，没问题。
> 

---

## 2）在 `index.md` 里把 bible 加进去（让首页能点到）

打开 `index.md` → Edit → 在你想放的位置加一行：

```markdown
## 📖 圣经与神学
- [Bible 学习](./bible.md)
- [笔记列表](./notes.md)

```

Commit。

✅ 现在首页就会出现 “Bible 学习” 入口，点进去就是 `bible.md`。

---

## 3）以后 bible 里面怎么继续扩展（推荐结构）

你有两种扩展方式，选一种就行：

### 方式 A：bible.md 直接列出所有具体笔记（最简单）

```markdown
# 📖 Bible 学习
- [2025-01-15 希伯来书 1](./2025-01-15-hebrews-ch1.md)
- [2025-01-22 希伯来书 2](./2025-01-22-hebrews-ch2.md)

```

### 方式 B：bible.md → 再分子栏目（更清晰，适合内容多）

- `bible.md` 只放分类入口
- 比如再建 `hebrews.md`，把希伯来书笔记都放那

`bible.md`：

```markdown
# 📖 Bible 学习
- [希伯来书](./hebrews.md)
- [传道书](./ecclesiastes.md)

```

`hebrews.md`：

```markdown
# 希伯来书
- [2025-01-15 第1章](./2025-01-15-hebrews-ch1.md)

```

---

## 4）一个小建议（让你以后不乱）

如果你笔记会越来越多，建议你最终用文件夹归档，比如：

```
/bible/
  hebrews/
    2025-01-15-ch1.md

```

但这不是现在必须做的，你已经能跑起来了，先保持简单也很好。
